Port Scan

Nmap​

/usr/share/nmap/scripts

SYN Scans: Avoid full handshake, too many connected ports = too much noise. But can take down an unstable network.

TCP Scans: Full three-way handshake and close connection.

UDP Scans: Destination Unreachable = closed, nothing = open | filtered

nmap tricks​

# noise level, 0 = paranoid, stealth | 5 = insane
# avoid 5, use 4 for ctfs, 3 to 0 in real engagements
-T<0-5>

# OS scan
nmap -sS -O 

# with traceroute
nmap -sS --traceroute 

# banner grabbing
nmap --script=banner 

# rate to send packages | min / max
--min-rate 5000

# 512 probes at a time | probes -> open connections
--min-parallelism 512 

#verbose & debug
nmap --reason
nmap -vv
nmap -dd

# outputs 
# -oA -> all 
# -oN -> nmap 
# -oG -> grepable, the best 
# -oX -> XML, for exporting results
nmap <flag>

Basic Port Scans​

# service and default scripts
nmap -sV -sC --top-ports <number> -vv <target>

# custom scripts
nmap -sV --scripts=<script> -vv <target> 

# fast SYN
nmap -sS --top-ports <number> -vv <target>

# TCP
nmap -sT --top-ports <number> -vv <target>

# UDP
nmap -sU --top-ports 20 -vv <target>

# full port range 0-65535
nmap -sS -p- <target>

DNS Discovery​

nmap -p 53 -vv <network>/24

Active Directory​

nmap -p 88,135,139,389,445 -sV -sC -iL hosts.txt
  • 88 Kerberos
  • 135 MS-RPC
  • 139 SMB/NetBIOS - LEGACY
  • 389 LDAP
  • 445 SMB
  • 464 Kerberos (kpasswd)

Firewall evasion tricks​

# fragmented, 8bytes | -ff, 16 bytes per packet
nmap -f 

# fragmented, custom amount, always do multiple 8's
nmap --mtu

# custom datalength
nmap --data-length 

# spoof port to look like DNS
nmap --source-port 53 

# 10 random ips for decoy
nmap -D RND:10 

# 2 chosen decoys, 1 real
nmap -D <ip>,<ip>,ME 

# 4 decoys, 2 set, 2 random, 1 real
nmap -D <ip>,<ip>,RND,RND,ME 

# need to be able to sniff traffic
nmap -Pn -e <net_interface> -S <spoofed>

Zombie Scan​

Zombie Scan needs an idle network device.

How it works:

  • Sends SYN/ACK to zombie, recieves RST and check the ip id heade
  • Sends SYN/ACK to the real target with the ip spoofed to the zombie ip.
  • If the port is closed, the target sends RST to zombie, and doesn't increment the zombie ip id.
  • If the port is open | filtered, the target sends SYN/ACK to the zombie, incrementing the zombie ip id.
  • Then, the scan checks again the ip header of the idle zombie.
  • If increment is equal 1 -> closed. If increment is equal 2 -> open | filtered
nmap -sI <zombie_ip>
