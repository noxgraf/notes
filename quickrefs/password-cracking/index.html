Password Cracking

Rainbow Tables:

  • Crackstation
  • Hashes

Rainbow Tables: Precomputed hashes in a database for fast search. Usually the "dehasher" web sites do this.

Rule-Based Attacks: Rule-based attacks take an existing wordlist and apply transformations to each word.

  • Capitalise the first letter: password → Password
  • Append a number: password → password1
  • Add a special character: password → password!
  • Substitute characters: password → p@ssw0rd

Mask Attacks: A mask attack is a structured Brute-force attack in which you define the pattern of the password rather than just the character set.

  • lowercase -> ?l
  • uppercase -> ?u
  • digits -> ?d
  • special -> ?s
  • ascii -> ?a
  • Ex. "Summer2026!" -> ?u?l?l?l?l?l?d?d?d?d?s

Tips:

  • No info? rockyou.txt -> Failed? rockyou.txt + best64
  • Known pattern? Mask attack
  • Short? Brute force (cruch)
  • Company-specific? Custom wordlist + rules
  • cupp is a good tool for generating custom wordlists based on user profile.

commands​

# john formats
john --list=formats

#john display prev cracked
john --show --format=raw-md5 hash.txt

# john md5
john --format=raw-md5 --wordlist=/usr/share/wordlists/rockyou.txt hash.txt

# john rule-based
john --format=raw-md5 --wordlist=/usr/share/wordlists/rockyou.txt --rules=wordlist hash.txt

# john rule-based
john --wordlist=base.txt --stdout --rule=haha22 >> output.txt

Hashcat Modes​

  • md5 -> 0
  • sha-1 -> 100
  • sha-256 -> 1400
  • sha-512 -> 1700
  • NTLM -> 1000
  • bcrypt -> 3200

commands​

# hashcat basic, -a -> attack mode, 0 = dictionary | 3 -> mask
hashcat -m <mode> -a 0 hash.txt /usr/share/wordlists/rockyou.txt --quiet

# hashcat mask, -a -> attack mode, 3 = mask attack
hashcat -m <mode> -a 3 hash.txt '?l?l?l?l?l?l?l?l'

# hashcat rule-based
hashcat -m <mode> -a 0 hash.txt /usr/share/wordlists/rockyou.txt -r /usr/local/hashcat/rules/best64.rule

# w/ output
hashcat -m 0 -a 0 hash.txt /usr/share/wordlists/rockyou.txt -o cracked.txt
hashcat -m <mode> hash.txt --show

# stop mid crack and restore after
hashcat -m <mode> -a 0 hashes.txt /usr/share/wordlists/rockyou.txt --session=example_crack
hashcat --session=example_crack --restore

A little hash theory​

Hashes

Can't be decrypted. It’s meant to be impossible (or computationally impractical) to go from the output back to the input. Good hashing algorithms will be relatively fast to compute and prohibitively slow to reverse. Any changes in the input (single bit) changes the output.

Hashing helps protect data’s integrity and ensure password confidentiality.

Store only the hash of a password, with a salt appended (salt prevents two passwords to have the same hash, and are unique to the user). The only way to find the original password is to hash a password and see if they match.

Hash collision is when 2 or more inputs have the same hash.

Password Hashes = hash(password + salt)

  • MD5: 32 hex chars
  • SHA-1: 40 hex chars
  • SHA-256: 64 hex chars
  • NTLM: 32 hex chars
  • bcrypt: ~60 hex chars
