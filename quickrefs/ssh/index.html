SSH

  • pub key -> server
  • priv key -> client, NEVER SHARE
  • authorized_keys -> pub keys of server + permissions
  • known_hosts -> saved fingerprint from servers accepted

Connect​

# normal
ssh user@<ip>

# with graphics, if X11 supported
ssh <user>@<ip> -X

# non standard port
ssh -p <port> <user>@<ip>

# ssh-key
ssh -i id_rsa <user>@<ip>

Add hostname for fast login​

~/.ssh/config

host webserver
    HostName <ip>
    User <user> 
    Port 22
    IdentityFile ~/.ssh/id_rsa

then: ssh webserver

SSH-Key​

# Generate an Ed25519 key
ssh-keygen -t ed25519

# For systems that don't support Ed25519, use RSA with 4096 bits
ssh-keygen -t rsa -b 4096

#copy to server
ssh-copy-id user@<ip>

Permissions​

chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys
chmod 600 ~/.ssh/id_rsa

Port Forwarding​

Local -> Opens locally and forwards traffic through SSH to the host and port on the other side.

Dynamic -> Sets up SOCKS proxy. Any tool that supports socks can send traffic through this proxy. Or just use ProxyChains. Any port, any host.

# single
ssh -L 8080:localhost:80 <user>@<compromised_host> -N

# dynamic
ssh -D 9050 <user>@<compromised_host> -N

Copy files​

# copy from remote to local
scp <user>@<ip>:/home/user/file.txt ~/ 

# copy from local to remote
scp file.txt <user>@<ip>:/home/user/

Execute without shell​

ssh <user>@<ip> "cat /etc/passwd"

Bruteforce​

hydra -l "user" -P /usr/share/wordlists/<wordlist> <ip> -t 4  ssh
