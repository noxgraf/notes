Docker

Docker: Uses namespaces to create an app environment

  • Image: Package with files, configs and everything needed to run a container.
  • Container: Process boxes to "contain" the application and its environment with a docker engine on top of the kernel of the OS.
  • Docker Compose: Allow multiple containers to interact with each other.
  • Dockerfiles: Instructions on how to assemble the image.

Docker Server uses sockets to communicate with the client. You can interact with the Docker Server using curl and stuff. This is a good attack vector if misconfigured, because you can remotely stop, start and access Docker containers, if you interact with the socket.

/var/run/docker.sock

  • List containers: curl http://<target>:2375/containers/json
  • Stop containers: curl -X POST http://<target>:2375/containers/<container>/stop
  • Create a rogue container: curl -X POST http://<target>:2375/containers/create

A dockerfile example:

FROM ubuntu:22.04

WORKDIR / 

RUN touch helloworld.txt

docker build -t helloworld .

An Apache Webserver dockerfile example:

FROM ubuntu:22.04

RUN apt update -y && apt install apache2 -y

EXPOSE 80 

CMD ["apache2ctl", "-D","FOREGROUND"]

docker run -d --name webserver -p 80:80 webserver

  • Each instruction run in its own layer, so chain your commands
# Before
RUN apt update -y

RUN apt upgrade -y

RUN apt install apache2 -y

# After
RUN apt update -y && apt upgrade -y && apt install apache2 -y

A docker-compose.yml example:

  • Run one web server and one database server.
version: '3.3'
services:
  web:
    build: ./web
    networks:
      - ecommerce
    ports:
      - '80:80'


  database:
    image: mysql:latest
    networks:
      - ecommerce
    environment:
      - MYSQL_DATABASE=ecommerce
      - MYSQL_ROOT_PASSWORD=helloword
    
networks:
  ecommerce:

Docker Basics​

# pull image
docker pull <image>:<tag>

# list images
docker image ls

# remove
docker image rm <image>:<tag>

# list running containers
docker ps

# list all containers
docker ps -a

# run with interactive shell
docker run -it <image> /bin/bash

# run with binded ports
docker run -p 80:80 <image>

# create network
docker network create <name>

# use network
docker run -p 80:80 --name <container_alias> --net <net_name> <image>

Compose​

# compose
# (re)create/build and start containers
docker compose up

# start only
docker compose start

# stop only
docker compose stop

# stop and delete containers
docker compose down

# build but dont start
docker compose build
