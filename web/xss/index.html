Cross-Site Scripting (XSS)

  • For callbacks: interact.sh
  • PayloadAllTheThings - XSS
  • PortSwigger

JavaScript code an attacker injects into a web application that runs in another user's browser.

Reflected: Non-Persistent. Web app takes user input (query string, form field, ...) and displays it on a page without sanitising it.

Stored: Persistent. Web app saves attacker-controlled input on the server and displays it to other users.

DOM-Based: Client-side JavaScript reads attacker-controllable data from the DOM (URL, location.hash, location.search, document.referrer, localStorage, etc.) and writes it back into the page in an unsafe way (e.g. innerHTML, document.write, eval).

Blind: Stored XSS but you can't see the payload working or test it against yourself.

  • A website has a contact form where you can message a member of staff. The message content isn’t checked for malicious code, allowing the attacker to enter anything they wish. These messages are then turned into support tickets, which staff view on a private web portal.
  • Ensure your playload has a callback like fetch()

Fix examples​

PHP Reflected​

Vulnerable:

<?php
$search_query = $_GET['q'];
echo "<p>You searched for: $search_query</p>"; // reflect user query
?>

Fix:

  • PHP Docs - htmlspecialchars()
  • htmlspecialchars()
<?php
$search_query = $_GET['q'];
$escaped_search_query = htmlspecialchars($search_query); // escape html special chars
echo "<p>You searched for: $escaped_search_query</p>";
?>

PHP Stored​

Vulnerable:

$comment = $_POST['comment'];
mysqli_query($conn, "INSERT INTO comments (comment) VALUES ('$comment')");

$result = mysqli_query($conn, "SELECT comment FROM comments");
while ($row = mysqli_fetch_assoc($result)) {
    echo $row['comment'];
}

Fix:

  • You shouldn't use raw sql, but mysqli_real_escape_string() escapes special chars for use in SQL statements.
    • PHP Docs - mysqli
  • Then, like reflected, htmlspecialchars() to display.
$comment = mysqli_real_escape_string($conn, $_POST['comment']);
mysqli_query($conn, "INSERT INTO comments (comment) VALUES ('$comment')");

$result = mysqli_query($conn, "SELECT comment FROM comments");
while ($row = mysqli_fetch_assoc($result)) {
    $sanitizedComment = htmlspecialchars($row['comment']); 
    echo $sanitizedComment;
}

Node.js Reflected​

Vulnerable:

const express = require('express');
const app = express();

app.get('/search', function(req, res) {
    var searchTerm = req.query.q;
    res.send('You searched for: ' + searchTerm);
});

app.listen(80);

Fix:

  • One of many external lib, npm, use it with caution and do your research.
  • sanitize-html library
  • sanitizeHtml()
const express = require('express');
const sanitizeHtml = require('sanitize-html'); 


const app = express();

app.get('/search', function(req, res) {
    const searchTerm = req.query.q;
    const sanitizedSearchTerm = sanitizeHtml(searchTerm); // removes unsafe attributes
    res.send('You searched for: ' + sanitizedSearchTerm);
});

app.listen(80);

Flask (Python) Reflected​

Vulnerable:

from flask import Flask, request

app = Flask(__name__)

@app.route("/search")
def home():
    query = request.args.get("q")
    return f"You searched for: {query}!"

if __name__ == "__main__":
    app.run(debug=True)

Fix:

  • html.escape()
  • Python Docs - html
  • Could also use MarkupSafe
from flask import Flask, request
from html import escape

app = Flask(__name__)

@app.route("/search")
def home():
    query = request.args.get("q")
    escaped_query = escape(query)
    return f"You searched for: {escaped_query}!"

if __name__ == "__main__":
    app.run(debug=True)

ASP.NET Reflected​

Vulnerable:

public void Page_Load(object sender, EventArgs e)
{
    var userInput = Request.QueryString["q"];
    Response.Write("User Input: " + userInput);
}

Fix:

  • HttpUtility.HtmlEncode()
  • Microsoft Learn
using System.Web;

public void Page_Load(object sender, EventArgs e)
{
    var userInput = Request.QueryString["q"];
    var encodedInput = HttpUtility.HtmlEncode(userInput);
    Response.Write("User Input: " + encodedInput);
}

Fix:

using System.Web;

public void SaveComment(string userComment)
{
    var command = new SqlCommand("INSERT INTO Comments (Comment) VALUES (@comment)", connection);
    command.Parameters.AddWithValue("@comment", userComment);
}

public void DisplayComments()
{
    var reader = new SqlCommand("SELECT Comment FROM Comments", connection).ExecuteReader();
    while (reader.Read())
    {
        var comment = reader["Comment"].ToString();
        var sanitizedComment = HttpUtility.HtmlEncode(comment); //here
        Response.Write(sanitizedComment);
    }
    reader.Close();
}
