Insecure Deserialization

  • PayloadAllTheThings

Application trusts serialized data and use it without proper validation of authenticity. Code review is the best way to look for it, but behavior analysis can lead to great results too. Appending a ~ to a PHP file can also get you access to backup or temporary files. If you don't have access to the code, exploiting Insecure Deserialization is severily deprioritized.

  • Serialization: Transforming an object's state into a human-readable or binary format. It's essential in applications where data must be transferred across a network.
  • Deserialization: Converting the formatted data back into an object.

Remember Log4j (CVE-2021-44228)? Insecure Deserialization.

Serialization:

<?php
$noteArray = array("title" => "note", "content" => "test note");
$note = serialize($noteArray); 
file_put_contents('note.txt', $note);
?>
//a:2:{s:5:"title";s:4:"note";s:7:"content";s:9:"test note";}

Deserialization:

<?php
$note = file_get_contents('note.txt'); 
$noteArray = unserialize($note); // if the user have control over the content here you get rekt
echo "Title: " . $noteArray['title'] . "<br>";
echo "Content: " . $noteArray['content'];
?>
  • .NET -> Serialization
  • PHP -> Serialize
  • Java -> Serializable
  • Python -> Pickle
  • Ruby -> Marshal
