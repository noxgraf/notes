Basics

/etc -> files used by the system
/var -> (variable data) frequently accessed or written by services and applications
/var/logs -> logs
/root -> root system user
/home -> users directories
/tmp -> temporary files

cron: a time-based job scheduler.
crontab: special file with formatting recognized by cron process.

  • Crontab generator
  • Crontab guru

crontab example:

# Runs backup of Documents, * is a wildcard (crontab -e nvim)

0 */12 * * * cp -R /home/moriarty/Documents /var/backups/

terminal​

  • This is too basic, but sometimes you get nervous.
# change dir
cd 

# list dir
ls -la 

# find file by name and redirect errors
find <location> -name <file_name> 2>/dev/null

# list partitions
df -h

# system info | cat /proc/version | cat /etc/os-release
uname -a

# set alias | unalias
alias ns="ls /usr/share/nmap/scripts/ | grep"

# set function | unset -f nscript
nscript() { ls /usr/share/nmap/scripts/ | grep "$@" } 

# processes | ps axjf
ps aux

# real time processes
top

# kill process | pkill -u <user>
kill <PID> 

# start, stop, enable, disable, status
systemctl <option> <service> 

# start in background | CTRL + Z
<command>& 

# focus back
fg 

# create cron tab
crontab -e <text_editor>

# network legacy
ifconfig

# network new
ip a s

# network routes | route
ip r

# arp table | arp -a 
ip n

# query installed packages and versions
dpkg -l
